Mac通过终端设置系统的睡眠模式及电池配置。

Mac 电源管理设置-pmset

[toc]

简单介绍

概念

pmset，操纵电源管理设置（power management setting）。

顾名思义，这个工具可以设置和列出电源管理设置，像空闲睡眠时间、访问时唤醒、断点重启等等。其中一部分功能在系统设置中通过图形界面的方式提供了，然而大部分更加灵活的选项并没有显示出来。

这种情况下，如果需要调整未提供的管理设置。就需要使用pmset命令来进行调整。

pmset命令必须使用root用户来运行。

查看对应相关内容可以在终端运行 man pmset， 查看官方说明。

相关标记

pmset 可以根据使用情况设置不同的电量管理设定。当设备正在充电、使用电池、UPS（不间断电源）或三者都有的情况下，可以应用不同的设定。下面是相关的标记：

  • -c 调节设定用于连接充电器的时候
  • -b 调节设定用于使用电池的时候
  • -u 调节设定用于使用 UPS 的时候
  • -a 调节设定用于全部情景

常用命令

  1. 显示信息

option 参数

  • -g (不带参数) 显示当前正在使用的设置

  • -g live 显示当前正在使用的设置（等同于不带参数）

  • -g custom 显示为所有电源的自定义设置（分别显示电池与电源设置）

  • -g cap 显示机器支持哪些电源功能

  • -g sched 显示计划启动、唤醒或关闭、睡眠事件

  • -g ps/batt 显示电池和UPS状态

  • -g pslog 显示电源（电池或UPS）状态的连续日志

  • -g rawlog 显示直接读取电池的电池状态的连续日志

  • -g therm 显示影响 CPU 速度的热力条件。不适用于全部平台

  • -g thermlog 显示影响 CPU 速度的热量的通知日志。不适用于全部平台

  • -g assertions 显示电量断言的概要。断言可以阻止系统睡眠或显示器睡眠。适用于 10.6 及更新版本

  • -g assertionslog 显示电量断言的创建及释出日志。适用于 10.6 及更新版本

  • -g sysload 显示“系统负载顾问”——。适用于 10.6 及更新版本

  • -g ac/adapter 显示关于交流电源适配器的详情。只有 MacBoook 和 MacBook Pro 支持。

  • -g log 显示睡眠，唤醒及其他电源管理时间的历史。这个日志只提供给管理员和调试目的

  • -g uuid 显示当前活跃的睡眠、唤醒的 UUID。

  • -g uuidlog 显示当前活跃的睡眠、唤醒的 UUID，并且打印一个新的被系统设置的 UUID

  • -g history 一个调试工具。当使用启动参数 io=0x3000000 被启用时，打印系统睡眠唤醒的 UUID 的时间线。

  • -g powerstate 打印当前为 IO Kit 驱动电源状态。调用者应该至少提供一个 IO Kit 类名来作为参数。如果没有类名被听过，它将打印全部驱动的电源状态。

  • -g powerstatelog

  • -g stats 打印自系统启动以来睡眠和唤醒系统的计数。

  • -g systemstate 打印当前系统的电量状态和可用能力。

  • -g everything 打印每个参数。这是一个有用的快速收集所有 pmset 提供的输出。适用于 10.8。

  1. 修改安全休眠级别

台式默认支持模式。睡眠时内存保持供电，其他部件断电，系统将不会备份内存到硬盘，唤醒时直接从内存恢复。

  • 优点是快速唤醒、睡眠，因为内存保持供电，无论唤醒还是睡眠都很快，节省SSD空间和寿命。因为内存数据不写入硬盘，所以硬盘里不会有内存镜像，也不会缩短SSD寿命。
  • 缺点是如果在睡眠过程中没电了的话，Mac会丢失内存中的数据，数据安全性不高。同时电力消耗最大，导致待机时间短。

笔记本默认支持模式。将内存镜像写入磁盘的同时也给内存供电，直到电池的临界值，系统会存储一份内存的备份到硬盘，唤醒时从内存唤醒，除非内存断电了将会由硬盘唤醒。相对其他模式来说，电力消耗一般，唤醒速度相对适中。

只能通过pmset设置。系统将内存镜像写入硬盘，写入后断开电脑所有部件断电，唤醒的时候从硬盘读取。

  • 优点是最省电，待机时间长，不怕数据丢失，对电池寿命友好。
  • 缺点是睡眠和唤醒速度最慢，也会缩短SSD寿命。

设置参数

参数

displaysleep: Mac闲置多长时间后进入显示器睡眠（值为分钟，或者设置 0 来禁用，不能长于sleep）。

disksleep: Mac闲置多长时间后关闭硬盘。这个系统偏好里也有，只不过换了一个字眼—如果可能，使硬盘进入睡眠—勾上这个的话系统就会自动根据sleep的时间设一个合适的时间（值为分钟，或者设置 0 来禁用，不能长于sleep）。

sleep: 系统睡眠计时器（值为分钟，或者设置 0 来禁用）。

注意：睡眠时间的设置上，应该是 sleep >= disksleep >=displaysleep

womp: 远程网络唤醒（值为 1 或者 0）

ring: 通过调制解调器环境（值为 1 或者 0）

hibernatemode: 改变休眠模式。请小心使用（值为整数）

hibernatefile: 改变休眠镜像文件位置。镜像应该只被定为到根卷中。请小心使用（值为路径）

powernap: 这个就是电能小憩，你可以在系统偏好设置里选择开或关，跟在这里设置是一样的，1是开启 0是

standby：使得机器在睡眠了一段指定的时间间隔后，核心电源管理会自动休眠机器。即当满足以下条件（接电源、没有外界设备、没有网络活动），到了standbydelay x秒后，就开始启动安全睡眠模式。根据highstandbythreshold设置的电量百分比，当满足对应条件standbydelayhigh或standbydelaylow时，进入安全睡眠模式，往硬盘上写入sleepimag后进入深度睡眠。

standbydelayhigh：设置电量高于highstandbythreshold设置的百分比时，进入睡眠的延迟，单位秒。

standbydelaylow：设置电量低于highstandbythreshold设置的百分比时，进入睡眠的延迟，单位秒。

networkoversleep: 这个设置影响 OS X 在系统睡眠时如何联网。这个设置不被全平台使用，不支持修改这个值。

ttyskeepawake: 当任何 tty（如：远程登录会话） 在活动状态时，阻止系统空闲睡眠。远程用户正在活动时防止Mac进入睡眠，1是开启 0是关闭。

tcpkeepalive

acwake: 当电源（AC 或者电池）改变的时候唤醒机器（值为 1 或者 0）

proximitywake：在受支持的系统上，该选项根据使用相同iCloud id的设备的接近程度控制系统从睡眠中唤醒。

halfdim: 显示器睡眠时使显示器亮度改变为当前亮度的一半，1是开启 0是关闭。如果关闭这个功能的话，显示器睡眠会直接关掉显示器。

lidwake: 当笔记本打开盖子的时候唤醒机器（值为 1 或者 0）

(以下部分通过 man pmset 查看)

autorestart: 当断电后自动重启（值为 1 或者 0）

lessbright: 显示休眠将使用介于全亮度和全亮度之间的中间半亮度状态(值为 1 或 0)

sms: 当重力突然改变时，使用瞬时运动传感器来停止磁盘头（值为 1 或者 0）

destroyfvkeyonstandby: 当变为待机模式时销毁文件库密钥。默认地，当系统待机时密钥被保留。如果关键文件被销毁，将导致当用户退出待机模式时输入密码（值为 1 – 销毁 ，0 – 保留）

命令用例

想要修改某一项设置的话，就打开终端，按照以下的格式输入：

  1. 修改网络状态, 开启, 盒盖即断网, 类似关机状态
  1. 合上屏幕之后，在4200分钟之内如果被唤醒，则从内存直接调用数据，如果4200分钟后依然未请求唤醒，系统则会移送所有内存数据到硬盘，并且停止供电。
  1. 禁止sleepimage的生成与写入（耗电量巨大，有数据丢失风险）
