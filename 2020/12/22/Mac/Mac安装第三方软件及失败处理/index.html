Mac安装第三方软件失败处理

发表于 2020-12-22 更新于 2021-05-18 分类于 Mac 阅读次数：
处理安装第三方软件过程中提示「打不开xxx，因为它来自身份不明开发者。」，或提示「xxx.app已损坏，打不开。你应该将它移到废纸篓」异常时修复处理。

处理第三方应用软件无法打开或文件损坏

在Mac安装第三方软件时可能会有以上提示，当遇到「打不开xxx，因为它来自身份不明开发者。」时，触发了苹果的安全机制，检测到软件没有签名，禁止随意安装。苹果默认只能安装 App Store 下载的或被认可的开发者开发的软件。

在遇到这种问题时，首先按住 Control 再次点击图标安装。如果还是安装失败，再执行以下操作

在执行以下操作前，请确认你要安装的软件时安全的。

打开系统偏好设置，安全性与隐私可以查看当前的允许的安装渠道。

此时可以看到系统默认提供的只有两种渠道。想要安装则需要一个隐藏渠道「任何来源」。

如果有「任何来源」，在点击左下角的「点按锁按钮以进行更改。」后，输入电脑解锁密码后，选择「任何来源」即可。

如果没有「任何来源」则执行以下操作。

Mac 允许任何来源

系统设置中隐私设置没有显示全部来源时，终端输入

sudo spctl --master-disable

输入完成后会要求输入电脑开机密码。在输入密码的过程中，密码不会有变动显示，输入完成直接回车。

如果按完回车没有任何报错信息则修改成功，重新进到系统偏好设置中的安全性与隐私，此时就有了「任何来源」。

再次安装第三方软件。

可能还是会失败。就是这么惨。。。

可能会报错「xx已损坏无法打开。你应该将它移到废纸篓」。如果遇到了继续往下看。

Mac 打开软件提示已损坏无法打开的解决办法

  1. 如题失败时，终端输入，不要！！回车。

sudo xattr -r -d com.apple.quarantine /Applications/xxxx.app

  1. 将 /Applications/xxxx.app 替换为你要打开app的名字，如果不会直接将这段删除，并执行第三步，如果会替换，跳过第三步
  2. 打开Finder，找到要打开的应用的路径，按住软件，拖动到终端里第一步输入的代码的后面
  3. 回车，并输入开机密码。密码不会有显示变动，凭感觉输入后，回车。如果没有报错则成功。
  4. 重新安装软件。
