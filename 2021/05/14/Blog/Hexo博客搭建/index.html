Hexo博客搭建

发表于 2021-05-14 更新于 2021-05-17 阅读次数：

根据 Hexo 官方文档及其他教学贴实现的 Hexo 搭建过程。因为是踩着各位巨人的肩膀，所以出现的错误较少，感谢各位前辈。

概述

Hexo：一个快速、简洁且高效的博客框架。基于 Node.js 运行，使用 Markdown 解析文章。

Hexo官方文档

GitHub Pages：由 GitHub 官方提供的一种免费的静态站点托管服务，托管和发布静态网站页面。

GitHub Pages 官方文档

原理

本地编写MD，通过 Hexo 解析，渲染成具有主体样式的 HTML 静态网页，最后发布到 GitHub。

环境搭建

  1. 安装Git（使用的Mac安装Xcode后自动安装），使用 git --version 判断是否安装及版本

    下载地址：https://git-scm.com/downloads

  2. 安装Node.js，使用 node -v 判断是否安装及版本。npm 默认随 Node.js 安装。

    下载地址：http://nodejs.cn/download/

  3. 安装 Hexo，终端执行 $ npm install -g hexo-cli

完成后，文件夹内文件结构为

_config.yml：网站的配置信息，在此配置大部分参数。

package.json：应用程序的信息。

scaffolds：模板文件夹。新建文章时，Hexo 会根据 scaffold 来建立文件。

source：资源文件夹。开头命名为_的文件/文件夹和隐藏的文件会被忽略。

Themes：主题文件夹。Hexo会根据主题来生成静态页面。

public：网站文件

添加SSH Key

  1. 检查是否存在SSH Key，终端执行

    如果有 id_rsa.pub 与 id_rsa 则跳过第二步。

  2. 生成SSH key，终端执行

    会在 ~/.ssh 目录下生成 id_rsa.pub 和 id_rsa 。

  3. Finder 前往 ~/.ssh 使用文本格式打开 id_rsa.pub ，并复制内容。

    进入 GitHub -> Setting -> SSH and GPG keys -> New SSH key

    Title 添加一个相关标题，将复制的内容贴到Key中，然后点击 Add SSH key。输入账号密码，添加完成后自动返回到列表界面，即可看到刚刚添加的SSH key。

绑定域名

  1. 万网购买域名，我在阿里购买的

  2. 登录阿里云，进入域名模块，点击已购买的域名后的解析，点击添加记录

  3. 记录类型选择 CNAME，主机记录输入 www，记录值填 <用户名>.github.io，保存。再添加一条记录，记录类型选默认为 A，主机记录填写 @，记录值填写git分配的ip(通过终端 ping 自己的 xx.github.io 获取)，保存。

  4. Hexo 的配置文件 _config.yml 中，配置 url 设置为 https://www.xx.xx 即已申请的地址。

  5. 登录 GitHub ，进入 Setting -> Pages ，设置 Custom domain 为 https://www.xx.xx 即已申请的地址。

    正确完成设置后，GitHub Pages 设置中可以看到

    此时再去访问域名就可以直接进入博客了。

  6. 如果每次重新部署完无法访问域名，则将 CNAME 文件创建到 Source 文件夹内。

添加阅读次数

  1. 申请 LeanCloud 账号并认证

  2. 点击左上角 LeanCloud 进入应用列表，并创建应用

  3. 点击应用，进入应用信息界面，点击左侧菜单栏的 数据存储 -> 结构化数据

  4. 点击 创建 Class，Class 名称 必须为 Counter，选择 无限制。（为了兼容及避免错误）

  5. 点击左侧菜单栏的 应用Keys，复制AppID及AppKey。

  6. 进入主题的配置文件中，搜索 leancloud_visitors，将 enable 改为 true，并粘贴上 AppID及AppKey。

  7. 为了确保安全性，在 LeanCloud 的左侧菜单栏中选择 安全中心，在 Web 安全域名 中填入我们的域名。

  8. 部署后可能会发现阅读次数未显示数字，而是 Counter not initialized! More info at console err msg.
