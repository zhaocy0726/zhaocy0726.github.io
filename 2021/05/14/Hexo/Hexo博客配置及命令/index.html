Hexo博客配置及命令

发表于 2021-05-14 更新于 2021-05-18 分类于 Blog 阅读次数： Valine：
由 Hexo 官方文档获取的常用的配置及命令。

配置

服务器配置

可以在 _config.yml 中修改大部分的配置。

官方文档： https://hexo.io/zh-cn/docs/configuration

文章配置

Front-matter 是文件最上方以 --- 分隔的区域，用于指定个别文件的变量。

常用预设参数

参数 描述
layout 布局
title 标题
date 创建日期
update 修改日期
comments 开启评论功能（默认 true）
tags 标签（没有顺序和层次）
categories 分类（具有顺序性和层次性）

命令

init - 新建网站

本命令相当于执行了以下几步：

  1. Git clone hexo-starter 和 hexo-theme-landscape 主题到当前目录或指定目录。
  2. 使用 Yarn 1、pnpm 或 npm 包管理器下载依赖（如有已安装多个，则列在前面的优先）。

new - 新建文章

如果没有 layout 的话，使用默认 layout 参数。如果标题有空格，需要用引号括起来。

默认情况下，会创建 source/_posts/test.md，md 中 title 为 test。对于独立页面来说，Hexo 会创建一个以标题为名字的目录，并在目录中放置一个 index.md 文件。你可以使用 --path 参数来覆盖上述行为、自行决定文件的目录：

以上 1 命令会创建一个 source/about/me.md 文件，同时md中的 title 为 "About me"。如果没有指定title，即 2 命令会创建 source/_posts/about/me.md ，同时 md 中的 title 为 page。

1 中的 page 为 Hexo 提供的默认布局，共有三种 post 、 page 、 draft。在创建这三种将会默认保存到不同路径，自定义的其他布局和 post 相同。

布局 路径
post / 自定义 source/_posts
page source
draft _drafts

草稿默认不会显示在页面中。可在执行时加上 --draft 参数，或是把 render_drafts 参数设为 true 来预览草稿。

generate - 生成静态文件

publish - 发表草稿

使用 draft 布局模板的默认被保存到 source/_drafts 文件夹内，使用 publish 将草稿移动到 source/_posts 文件夹。

server - 启动服务器

deploy - 部署网站

render - 渲染文件

clean - 清除缓存文件

清除缓存文件 (db.json) 和已生成的静态文件 (public)。

在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

list - 列出网站资料

version - 显示 Hexo 版本
